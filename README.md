# ВТБ-Сообщества

## Описание

Приложение представляет собой платформу для взаимодействия клиентов банка ВТБ с сообществами на основе их транзакций. Клиенты получают персонализированные выгодные предложения и кэшбэки, а сообщества — доступ к целевой аудитории. Система использует комбинацию реляционной (PostgreSQL) и графовой (Neo4j) баз данных для анализа транзакций и управления социальной логикой.

**Функционал включает:**

- **Сообщества**: просмотр, вступление, категоризация по MCC-кодам транзакций; условия вступления основаны на количестве операций в категории (≥3)
- **Посты и комментарии**: создание, редактирование, удаление; лайки; вложенные комментарии (ответы); доступно только участникам сообществ
- **Кэшбэки и выгоды**: автоматическое начисление после совершения ≥3 операций в MCC-категории; специальные предложения сообществ с процентами выгоды
- **Рекомендации**: персонализированные предложения мест на основе истории транзакций из графовой базы данных
- **Разграничение прав**: аутентификация через JWT; пользователи управляют своими постами и комментариями

## Предметная область

Система предназначена для анализа банковских транзакций клиентов, предоставления доступа к сообществам на основе паттернов покупок, управления постами и комментариями, а также начисления кэшбэков.

**Основные сущности:**
- Пользователи (клиенты банка)
- Сообщества (тематические группы по категориям MCC)
- Посты и комментарии (контент внутри сообществ)
- Кэшбэки и выгоды (финансовые стимулы)
- Транзакции (хранятся в графовой БД Neo4j)

Система обеспечивает автоматический доступ к привилегиям на основе поведенческих паттернов (транзакций) и полный цикл социального взаимодействия внутри сообществ.

## Данные

### Реляционная база данных (PostgreSQL)

#### Пользователь (`client`)
- **id** (SERIAL, PRIMARY KEY) — уникальный идентификатор
- **first_name** (VARCHAR(50)) — имя
- **last_name** (VARCHAR(50)) — фамилия
- **login** (VARCHAR(50), UNIQUE) — логин для входа
- **password** (VARCHAR(50)) — хэшированный пароль
- **created_at** (TIMESTAMP, DEFAULT NOW()) — дата регистрации

#### Сообщество (`community`)
- **id** (SERIAL, PRIMARY KEY) — идентификатор сообщества
- **name** (VARCHAR(100)) — название
- **description** (TEXT) — описание
- **category_key** (VARCHAR(64)) — ключ категории (связь с MCC)
- **min_transactions** (INTEGER, DEFAULT 0) — минимальное количество транзакций для вступления
- **created_at** (TIMESTAMP, DEFAULT NOW()) — дата создания

**Примеры категорий:** `cafe`, `supermarket`, `fitness`, `pharmacy`, `electronics`, `fuel`

#### Пост (`post`)
- **id** (SERIAL, PRIMARY KEY)
- **id_sender** (INT, REFERENCES client(id)) — автор
- **id_community** (INT, REFERENCES community(id)) — сообщество
- **title** (VARCHAR(225)) — заголовок
- **text** (TEXT) — текст
- **rating** (INT, DEFAULT 0) — количество лайков
- **created_at** (TIMESTAMP, DEFAULT NOW())

#### Комментарий (`comment`)
- **id** (SERIAL, PRIMARY KEY)
- **id_post** (INT, REFERENCES post(id)) — пост
- **id_sender** (INT, REFERENCES client(id)) — автор
- **id_parent** (INT, REFERENCES comment(id)) — родительский комментарий (для ответов, NULL — корневой)
- **message** (TEXT) — текст
- **created_at** (TIMESTAMP, DEFAULT NOW())

#### Лайк поста (`post_like`)
- **id_client** (INT, REFERENCES client(id))
- **id_post** (INT, REFERENCES post(id))
- **created_at** (TIMESTAMP)
- **PRIMARY KEY** (id_client, id_post) — один лайк на пост

#### Кэшбэк (`cashback`)
- **id** (SERIAL, PRIMARY KEY)
- **amount** (FLOAT) — сумма кэшбэка
- **place** (INT) — MCC-код места
- **category_key** (VARCHAR(64)) — ключ категории
- **created_at** (TIMESTAMP, DEFAULT NOW())

#### Начисленные кэшбэки (`client_cashback`)
- **id_client** (INT, REFERENCES client(id))
- **id_cashback** (INT, REFERENCES cashback(id))
- **created_at** (TIMESTAMP, DEFAULT NOW())
- **UNIQUE** (id_client, id_cashback) — однократное начисление

#### Предложение выгоды (`cashback_offer`)
- **id** (SERIAL, PRIMARY KEY)
- **id_community** (INT, REFERENCES community(id))
- **title** (VARCHAR(200)) — название предложения
- **percent** (INT) — процент выгоды
- **description** (TEXT) — описание
- **created_at** (TIMESTAMP, DEFAULT NOW())

### Графовая база данных (Neo4j)

Графовая БД предназначена для хранения и анализа транзакций пользователей. Модель построена на принципе **одна транзакция = одна связь**, что обеспечивает детализированный анализ паттернов покупок.

#### Узлы (Nodes)

**User** — соответствует `client.id` из PostgreSQL
- `id` (int) — идентификатор пользователя
- `first_name` (str) — имя
- `last_name` (str) — фамилия

**Place** — торговая точка
- `name` (str) — название места
- `mcc` (int) — MCC-код (Merchant Category Code)
- `category_key` (str) — ключ категории (англ.)
- `category` (str) — название категории на русском
- `cashback_min` (int) — минимальный кэшбэк для категории (%)
- `cashback_max` (int) — максимальный кэшбэк для категории (%)

#### Связи (Relationships)

**HAS_TRANSACTION** — соединяет пользователя с местом транзакции
- `tx_id` (str) — уникальный идентификатор транзакции
- `amount` (float) — сумма в рублях
- `currency` (str) — валюта ('RUB')

**Направление:** `(User) -[:HAS_TRANSACTION]-> (Place)`

### Интеграция PostgreSQL и Neo4j

| Реляционная БД (PostgreSQL) | Графовая БД (Neo4j) | Назначение |
|----------------------------|---------------------|------------|
| `client` | `User` | Пользователи (ID синхронизированы) |
| `cashback.place` (MCC) | `Place.mcc` | Категории мест |
| — | `HAS_TRANSACTION` | Транзакции пользователей |
| `community.category_key` | `Place.category_key` | Связь сообществ с категориями |

**Ключевая логика:**
- Количество операций пользователя в категории проверяется в Neo4j (`neo4j_user_category_operations`)
- Условие вступления в сообщество — ≥3 операций в категории
- Разблокировка кэшбэков — ≥3 операций в соответствующей MCC-категории

## Общие ограничения целостности

- **Вступление в сообщество**: проверяется через Neo4j количество операций пользователя в категории сообщества; требуется минимум 3 операции
- **Кэшбэки**: доступны после совершения ≥3 операций в соответствующей MCC-категории; один пользователь может получить каждый кэшбэк только один раз
- **Посты и комментарии**: только участники сообщества могут создавать посты; комментарии поддерживают вложенность (ответы); пользователи могут редактировать/удалять только свои посты и комментарии
- **Лайки**: пользователь может поставить только один лайк на пост; повторный клик удаляет лайк

## Пользовательские роли

Система использует JWT-аутентификацию. Роли определяются на основе аутентификации:

### Аутентифицированный пользователь
- Просмотр ленты постов из своих сообществ
- Просмотр и вступление в сообщества (с проверкой условий через Neo4j)
- Создание, редактирование, удаление своих постов и комментариев
- Лайк/дизлайк постов
- Просмотр начисленных кэшбэков и доступных возможностей
- Просмотр персонализированных рекомендаций

### Неаутентифицированный пользователь
- Доступ только к экрану входа

## UI / API

### Мобильное приложение (Flutter)

- **Экран входа** — аутентификация по логину/паролю
- **Лента постов** — посты из сообществ пользователя с возможностью лайков и комментариев
- **Сообщества** — список сообществ с разделением на «Ваши», «Можете вступить», «Недоступные»
- **Детальная страница сообщества** — посты сообщества, кнопка создания поста (для участников)
- **Кэшбэки и выгоды** — начисленные кэшбэки и доступные предложения сообществ
- **Создание/редактирование поста** — форма с заголовком и текстом
- **Комментарии** — древовидная структура, возможность ответа, редактирования и удаления
